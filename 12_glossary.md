# Glossary

## Contents

The glossary contains the most important domain and technical terms that stakeholders use when discussing the system.

It can also be used as a source for translations when working in multilingual teams.

## Motivation

Terms should be clearly defined so that all stakeholders:

- have an identical understanding of these terms
- do not use synonyms and homonyms incorrectly

## Form

Use a table with the columns **Term** and **Definition**.

Additional columns can be added if translations are needed.

## Further Information

See the arc42 documentation: [Glossary](https://docs.arc42.org/section-12/)

## Terms

| Term                             | Definition                                                                                                                                          |
| -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Cloud Infrastructure (AWS/Azure) | A scalable hosting environment used to deploy and run the University Management System in a cloud environment such as AWS or Azure.                 |
| Financial Management System      | An external university system responsible for handling billing and financial processes, integrated with the University Management System.           |
| MongoDB                          | A NoSQL database used to store structured and semi-structured data such as students, courses, grades, and attendance records.                       |
| Node.js Backend                  | The server-side application responsible for business logic, API handling, authentication, and communication with the database and external systems. |
| React Web Client                 | The frontend application built with React that runs in the user’s browser and provides the user interface of the University Management System.      |
| REST API                         | A standardized communication interface using HTTP requests that enables interaction between frontend, backend, and external systems.                |
| Student Information System (SIS) | An existing external university system that manages student-related administrative data and is integrated into the University Management System.    |
